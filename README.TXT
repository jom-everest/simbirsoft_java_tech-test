1. Запуск программы.

Для того чтобы запустить программу необходимо перейти в папку "dist" и в консоли набрать 
java -jar "SummaryHtmlCreator.jar" "filename"
где "filename" - имя файла, содержащий данные по соискателю в формате "ключ" = "значение", 
причем имя файла указывается не полное, без расширения ".properties". Можно не указывать
 "filename", в этом случае в качестве ".properties" файла будет использовано "bobunov.properties"

2. Руководство пользователя.

  Программа "SummaryHtmlCreator.jar" предназначена для формирования резюме соискателя в виде html файла на основе данных по этому соискателю, размещенные в "properties" файле в формате "ключ" = "значение". Приложение создает html файл с названием "summary_{subjectName}.html", где "{subjectName}" это имя "properties" файла без расширения ".properties". Приложению на вход подается "properties" файл, например "bobunov.properties", где в формате "ключ" = "значение" записаны данные по данному соискателю, и которые потом будут использованы для формирования html файла.
  
  Ключи "properties", которые будут использованы:
  
  fio - фамилия имя отчество
  dob - дата рождения
  email
  skype
  avatar - ссылка на изображение соискателя
  phone - номер телефона
  
  Все нижеперечисленные ключи могут иметь несколько значений, т.е. в файле "properties" за ключом можно перечислить несколько значений этого ключа, при этом все значения должны заключены в скобки '"' и должны отделяться друг от друга запятой, пример: hobbies = "игра на барабане", "просмотр телека" , "тренажерный зал"
  В отличие от нижеперечисленных ключей, все вышеперечисленные ключи будут иметь только 1 значение самое первое указанное после ключа, все прочие отброшены
  
  education - образование
  add_education - дополнительное, образование, курсы
  experience - опыт работы
  hobbies - занятия для души и тела вне работы
  target - цели
  skills - профессиональные навыки
  
  Все прочие названия ключей не будут учтены
  
  Примечание: если в "properties" файле присутствуют несколько строк с одними и теми же названиями ключей, то будет учтена самая верхняя строка, все прочие проигнорированы.
